# 13 — PR Submission & Review Feedback Loop

## Goal

When all validation gates pass, the agent creates a structured, evidence-rich PR. When a human reviewer leaves comments, the orchestrator picks them up and triggers a re-work cycle.

## Current State

- No automated PR creation from the coding-agent backend
- GitHub CLI (`gh`) is available in the Nix environment
- The existing deployment workflow (`.github/workflows/deploy-mac-mini.yml`) handles CD

## Target State

- Agent submits PRs with structured descriptions including test results, screenshots, accessibility audit, performance metrics
- Orchestrator detects new PR review comments (webhook or polling)
- Review comments trigger re-work tasks
- Agent pushes new commits and updates the PR
- Loop continues until human approves or task is manually closed

## Implementation Steps

### Step 1: Create PR Manager Service

Create `src/core/pr-manager.service.ts`:

```typescript
@Injectable()
export class PrManagerService {
  constructor(
    private taskStateService: TaskStateService,
    private transcriptWriter: TranscriptWriterService,
  ) {}

  async createPR(task: Task): Promise<number> {
    const body = await this.buildPRBody(task);
    const title = `[THE Dev Team] ${task.title}`;

    // Use gh CLI to create the PR
    const result = await exec(
      `gh pr create --title "${title}" --body "${body}" --base main --head ${task.branch}`,
      { cwd: task.worktreePath },
    );

    // Parse PR number from output
    const prNumber = this.parsePRNumber(result);

    this.transcriptWriter.logEvent(task.id, {
      type: 'pr_created',
      prNumber,
      branch: task.branch,
    });

    return prNumber;
  }

  async buildPRBody(task: Task): Promise<string> {
    const gateResults = await this.taskStateService.getAllGateResults(task.id);
    const screenshots = await this.getScreenshots(task.id);

    return `## Summary
${task.description}

## Changes
${await this.getChangeSummary(task)}

## Validation Results

### Tests
${this.formatTestResults(gateResults)}

### Accessibility
${this.formatAccessibilityResults(gateResults)}

### Performance
${this.formatPerformanceResults(gateResults)}

### Design Review
${this.formatDesignResults(gateResults)}

## Screenshots
${screenshots.map(s => `### ${s.breakpoint}\n![${s.breakpoint}](${s.path})`).join('\n\n')}

## Environment
- Branch: \`${task.branch}\`
- Task ID: \`${task.id}\`
- Total Cost: $${task.cost.toFixed(2)}

---
*Generated by THE Dev Team*
`;
  }

  private async getChangeSummary(task: Task): Promise<string> {
    const diff = await exec(
      `git diff --stat main...HEAD`,
      { cwd: task.worktreePath },
    );
    return diff;
  }

  private formatTestResults(results: GateResult[]): string {
    const unitTests = results.find(r => r.gate === 'unit-tests');
    const integrationTests = results.find(r => r.gate === 'integration-tests');
    const e2eTests = results.find(r => r.gate === 'e2e-tests');

    return [
      unitTests ? `- Unit tests: ${unitTests.passed ? 'passed' : 'failed'}` : '',
      integrationTests ? `- Integration tests: ${integrationTests.passed ? 'passed' : 'failed'}` : '',
      e2eTests ? `- E2E tests: ${e2eTests.passed ? 'passed' : 'failed'}` : '',
    ].filter(Boolean).join('\n');
  }
}
```

### Step 2: Implement PR Review Feedback Detection

Create a polling mechanism (or webhook handler) that detects new review comments:

```typescript
@Injectable()
export class PrReviewWatcherService {
  constructor(
    private taskIntake: TaskIntakeService,
    private transcriptWriter: TranscriptWriterService,
  ) {}

  @Cron('*/2 * * * *')  // Check every 2 minutes
  async checkForReviewComments(): Promise<void> {
    // Get all open PRs created by the bot
    const prs = await this.getOpenBotPRs();

    for (const pr of prs) {
      const newComments = await this.getUnprocessedComments(pr.number);
      if (newComments.length > 0) {
        await this.createReworkTask(pr, newComments);
      }
    }
  }

  private async getOpenBotPRs(): Promise<PR[]> {
    const result = await exec(
      `gh pr list --author the-dev-team-bot --state open --json number,title,headRefName`,
    );
    return JSON.parse(result);
  }

  private async getUnprocessedComments(prNumber: number): Promise<ReviewComment[]> {
    const result = await exec(
      `gh api repos/{owner}/{repo}/pulls/${prNumber}/comments`,
    );
    const comments = JSON.parse(result);

    // Filter to comments we haven't processed yet
    // Track processed comment IDs in task state
    return comments.filter((c: any) => !this.isProcessed(c.id));
  }

  private async createReworkTask(pr: PR, comments: ReviewComment[]): Promise<void> {
    const commentSummary = comments
      .map(c => `- **${c.path}:${c.line}**: ${c.body}`)
      .join('\n');

    await this.taskIntake.submitTask({
      title: `Address PR #${pr.number} review feedback`,
      description: `Review comments on PR #${pr.number}:\n\n${commentSummary}\n\nAddress all comments, push new commits, and update the PR.`,
      source: 'pr_feedback',
      sourceRef: String(pr.number),
      branch: pr.headRefName,     // Reuse the same branch
    });
  }
}
```

### Step 3: Handle Re-work Tasks

When a re-work task is executed, it differs from a normal task:
- Uses the existing branch (doesn't create a new one)
- May reuse or recreate the sandbox environment
- The prompt includes review comments as context
- After fixing, pushes new commits to the same branch
- Updates the PR with a comment summarizing what changed

```typescript
// In the execution loop, detect re-work tasks:
if (task.source === 'pr_feedback') {
  // Don't create a new worktree — check out the existing branch
  const worktreePath = await this.gitService.createWorktree(task.id, task.branch, { useExisting: true });
  task.worktreePath = worktreePath;

  // Run implementer with review feedback as context
  await this.runRole('implementer', task, {
    prompt: task.description,  // Contains the review comments
    skills: ['execute'],
  });

  // Re-run validation
  await this.buildAndDeploy(task);
  await this.test(task);

  // Push and comment on PR
  await this.gitService.push(task.branch);
  await this.prManager.commentOnPR(task.prNumber!, 'Addressed review feedback. Updated validation results attached.');
}
```

async updatePRDescription(prNumber: number, task: Task): Promise<void> {
  const body = await this.buildPRBody(task);
  await exec(
    `gh pr edit ${prNumber} --body "${body}"`,
  );
}

### Step 5: Handle Approval and Merge

The agent does NOT merge PRs. Merging is the human's job. But the agent can:
- Detect when a PR is approved (via polling or webhook)
- Trigger cleanup of the sandbox environment
- Mark the task as completed

```typescript
@Cron('*/5 * * * *')
async checkForApprovals(): Promise<void> {
  const prs = await this.getOpenBotPRs();
  for (const pr of prs) {
    const reviews = await exec(`gh pr view ${pr.number} --json reviews`);
    const approved = JSON.parse(reviews).reviews?.some(
      (r: any) => r.state === 'APPROVED'
    );
    if (approved) {
      // Cleanup environment, mark task complete
      await this.cleanup(pr.taskId);
    }
  }
}
```

## Open Questions

- **Webhook vs polling:** Webhooks are faster but require exposing an endpoint. Polling is simpler but has a 2-minute delay. Start with polling; add webhooks later if needed.
- **Screenshot upload:** GitHub PR descriptions support images, but they need to be uploaded. Options: commit screenshots to the branch, use GitHub's image upload API, or host on a CDN.
- **Review comment threading:** GitHub has both PR comments and review comments (line-level). The watcher needs to handle both types.
- **Multiple rounds:** If a reviewer leaves comments, agent fixes, reviewer leaves more comments — this could loop indefinitely. Should there be a max re-work iterations limit?
